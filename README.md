# Теневой оркестр (Shadow Orchestra)

Интерактивный музыкальный перформанс, где участники управляют звуком движением телефона в воздухе. Киберпанк-ритуал в тёмной комнате с проектором, неоновым светом и живой музыкой.

---

## Что это

Веб-приложение, превращающее смартфоны в музыкальные инструменты. Участники открывают ссылку в браузере, наклоняют телефон — и рождается звук. Все телефоны синхронизируются в реальном времени. Дирижёр управляет темпом и тональностью. Проектор рисует визуализацию, реагирующую на каждого игрока.

**Два режима работы:**
- 🎹 **Синтезатор** — телефоны сами синтезируют звук по ролям
- 🎧 **Ремикс** — загружаем трек, телефоны крутят эффекты в реальном времени

Никаких приложений. Никаких установок. Только браузер и движение.

---

## Режимы работы

| Режим | Источник звука | Что делают телефоны | Визуализация |
|-------|---------------|---------------------|--------------|
| 🎹 Синтезатор | Синтезаторы на телефонах | Играют ноты по ролям (bass/pad/lead/arp/fx) | Орбиты, частицы, волны, пульсирующий метроном |
| 🎧 Ремикс | Трек на хосте (ноутбук с колонками) | Управляют эффектами (filter/reverb/delay/pitch/distortion) | Спектрограмма трека + искажения от эффектов |

Переключение между режимами — на пульте дирижёра. Визуализация переходит плавно (кроссфейд 500 мс).

## Шоу (3 минуты, режим Синтезатор)

| Минута | Режим | BPM | Тональность | Что происходит |
|--------|-------|-----|-------------|----------------|
| 0:00–1:00 | Ambient | 60 | C minor | Медленные движения, тягучий эмбиент, синие волны |
| 1:00–2:00 | Beat | 120 | C minor | Бас вступает, арпеджиатор разгоняется, частицы ускоряются |
| 2:00–2:45 | Chaos | 160 | C minor | Тряска телефонами, визуальный шторм, кульминация |
| 2:45–3:00 | Ambient | 40 | C minor | Затухание, финальная нота, тишина |

Свет выключен. Работают только экраны телефонов и проектор.

## Как это работает

1. Участник открывает ссылку на телефоне и получает роль.
2. Наклоняет телефон — гироскоп отправляет данные на сервер.
3. Сервер преобразует движение в звук (ноту или параметр эффекта) и отправляет обратно.
4. Телефон синтезирует звук через Tone.js (режим Синтезатор) или управляет эффектом на хосте (режим Ремикс).
5. Проектор получает данные всех игроков и рисует визуализацию.
6. Дирижёр со своего пульта меняет темп, тональность и переключает режимы.

## Технологии

| Слой | Стек | Зачем |
|------|------|-------|
| Сервер | Node.js + Express + Socket.IO | Синхронизация в реальном времени |
| Звук | Tone.js | Синтез звука и обработка эффектов в браузере |
| Сенсоры | DeviceOrientation API | Гироскоп телефона |
| Визуализация | p5.js | Орбиты, частицы, спектрограмма |
| Туннель | localhost.run / Cloudflare Tunnel | HTTPS для iOS Safari |

## Роли в команде

- 🎼 **Дирижёр** — ведёт пульт, управляет темпом, тональностью и переключает режимы по сценарию.
- 🎹 **Звуковой дизайнер** — настраивает синтезаторы, реверб, тембры и цепочку эффектов.
- 🤸 **Хореограф движений** — связывает жесты с музыкой: какой наклон = какая нота или параметр.
- 🌌 **Визуальный художник** — создаёт графику в p5.js для обоих режимов.
- 🔧 **Технический координатор** — поднимает сервер, туннель, помогает с подключением.

## Режим «Синтезатор»

Телефоны сами синтезируют звук. У каждого своя роль и характер звучания.

| Роль | Тип синтезатора | Как играет |
|------|-----------------|------------|
| bass | MembraneSynth, треугольная волна | Каждую четверть |
| pad | Synth, синусоида | По движению (длинные ноты) |
| lead | Synth, пилообразная | По движению (острое соло) |
| arp | Synth, квадратная | Каждую восьмую (ритмично) |
| fx | Synth, треугольная | По движению (размытый пэд) |

Общий реверб: decay 3 сек, wet 0.5.

## Режим «Ремикс»

Загружаем трек — телефоны становятся пультами эффектов. Трек играет на ноутбуке дирижёра через колонки. Участники своими движениями меняют звук в реальном времени.

### Роли эффектов

| Роль | Эффект | Параметр | Жест | Что делает со звуком |
|------|--------|----------|------|---------------------|
| filter | Частота среза | 200–8000 Hz | Наклон влево-вправо | Открывает/закрывает фильтр |
| reverb | Глубина реверба | roomSize 0–1 | Наклон вперёд-назад | Добавляет пространство |
| delay | Повторы | feedback 0–0.8 | Тряска | Создаёт эхо |
| pitch | Высота тона | -12..+12 полутонов | Поворот телефона | Сдвигает тон трека |
| distortion | Перегруз | amount 0–1 | Тряска | Добавляет грязь и насыщение |

### Визуализация ремикса

- **Центр:** живая спектрограмма трека (32 частотные полосы)
- **Вокруг:** 5 слотов эффектов на фиксированных позициях
- **Размер кружка:** зависит от того, насколько сильно игрок выкрутил эффект
- **Искажения спектра:** фильтр гасит частоты, реверб размывает, дилей создаёт эхо-копии, питч сдвигает спектр, дисторшн добавляет шум
- **Пустые роли:** отображаются тусклым плейсхолдером
- **Несколько телефонов на роль:** значения усредняются в один слот

## Технические решения

- **playMode vs mode:** `playMode` (synth/remix) — выбор источника звука. `mode` (ambient/beat/chaos) — визуальное поведение. Две независимые оси, чтобы смена визуального режима не ломала звук.
- **Tone.Freeverb вместо Tone.Reverb:** roomSize меняется плавно без асинхронной перегенерации импульсного отклика — критично при непрерывном управлении гироскопом.
- **Спектр через Tone.Analyser:** 32 полосы FFT на выходе цепочки эффектов, значения 0–255, отправка отдельным событием `spectrum-data` 10 раз/сек.
- **Кроссфейд визуализации:** при переключении режимов обе сцены рисуются одновременно с плавным переходом альфа-канала за 500 мс.
- **Загрузка треков:** файлы сохраняются на диск в `public/tracks/`, хост загружает их по HTTP URL. Никакого base64 через WebSocket.

## Важно для iOS

Safari блокирует гироскоп и Web Audio API без HTTPS и явного жеста пользователя.

**Решение:**
- **HTTPS** через localhost.run или Cloudflare Tunnel (без этого ничего не работает)
- **Кнопка «Активировать»** при загрузке player.html: сначала `DeviceOrientationEvent.requestPermission()`, потом `Tone.start()`. Порядок критичен — на iOS разрешение датчиков должно быть первым после касания.

### Правила вайб-кодинга

- **Никаких лекций** — только практика
- **Ошибки = фичи** — рассинхрон = полиритмия, странный звук = экспериментальный тембр
- **Движение важнее кода** — участники управляют звуком телом
- **Шоу как цель** — мы пишем код чтобы выступить
